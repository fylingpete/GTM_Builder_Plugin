# People Analyzer™ Grid

> **Status:** empty_template — drafted nach Core Values + Acc Chart, refresht quarterly + bei jedem Hire/Fire/Promotion
> **Team-required:** no (manager-driven, reviewed im Quarterly People Review)
> **Source:** EOS™ (Gino Wickman) — People Component, "Right Person + Right Seat"

> **STRICT TWO-SCORING-SYSTEMS.** Core Values columns nutzen `+ / +/- / −`
> (three-symbol, fuzzy ist erlaubt, weil Behaviors über Monate observed
> werden). GWC™ columns nutzen `Y / N` only — **kein `+/-` allowed**.
> "Mostly" auf seat-fit ist genau die Disease, die das Tool kuriert.
> 2-of-3 auf GWC™ ist nicht Partial-Pass, ist Fail.

## How to use this file

People Analyzer™ ist der simpler Tool im People Component — und einer
der mächtigsten, weil er jede People-Decision gegen explicite Criteria
zwingt. Ein Grid: Menschen in Rows, Criteria in Columns. Linke Hälfte
ist Right-Person (Core Values). Rechte Hälfte ist Right-Seat (GWC™).

Sobald gefüllt: Arguments stopen "über Feelings" und werden Score-vs-
Bar. Eine Person below-Bar triggert die Three-Strike-Rule (3 strukturierte
Gespräche über ~90 Tage), nicht sofortige Termination. Most People rise
above the Bar binnen Strike 1-2, oder self-select out vor Strike 3 weil
der Mismatch obvious wird.

## Rules (from the substance rubric)

- **two_scoring_systems (non-negotiable):** Core Values = `+ / +/- / −`;
  GWC™ = `Y / N` only. NIEMALS `+/-` in GWC™ columns.
- **bar_set_once:** Bar wird im Leadership-Team einmal gesetzt und
  konsistent über die ganze Org applied. Wickman's Default für 5 Core
  Values: 3 `+`, 2 `+/-`, 0 `−`, plus `Y` auf allen 3 GWC™.
- **gwc_binary:** Y oder N. 2-of-3 = Fail. Toleranz für Ambiguität auf
  GWC™ ist single most common reason EOS™ stops working.
- **continuous_not_annual:** Grid wird laufend gepflegt — quarterly
  reviews, jeder Hire, jeder Fire, jede Promotion. NICHT ein einmaliger
  Annual-Performance-Review.
- **below_bar_triggers_three_strike:** Below-Bar → Three-Strike-Rule
  (3 Gespräche über 90 Tage). NIEMALS sofortige Termination ohne
  Three-Strike-Prozess.
- **n_on_capacity_is_seat_not_person:** Capacity-N bedeutet diese Person
  × dieser Seat funktioniert nicht — fast immer ist die richtige Move
  ein anderer Seat, nicht Termination.

## Common Failure-Shapes

<!-- INSTRUCTION: Wenn ein Failure-Shape diagnosed wird, notieren. Hilft bei Pattern-Erkennung über Quartale. -->

- **"Wants it" but doesn't "Get it"** — enthusiastic-but-confused.
  Fix is selten Training; usually move them.
- **"Gets it" but "Doesn't want it"** — capable but disengaging.
  Find a Seat they actually want, oder part ways.
- **"Gets it and Wants it" but lacks Capacity** — hardest because
  everyone is on the same side. Different Seat where Capacity matches.
