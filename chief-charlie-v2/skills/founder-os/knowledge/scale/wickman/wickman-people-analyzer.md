## The People Analyzer™ — EOS™ (Gino Wickman)

The People Analyzer™ is the simplest tool in EOS™'s People Component — and one of the most powerful because it forces every people decision to be made against the same explicit criteria. The tool is a single grid: people in rows, evaluation criteria in columns. Half the columns evaluate "right person" fit (Core Values). The other half evaluate "right seat" fit (GWC™ — Get it, Want it, Capacity).

The tool's value is not analytical sophistication. There is none. Its value is making subjective people decisions visible and comparable. Once the grid is filled, arguments stop being about feelings and become about scores against criteria.

### Two scoring systems on one grid

The most important and most easily missed feature: **two different scoring systems on the same grid**. Conflating them collapses the tool.

#### Left side — Core Values columns (three-symbol scoring)

One column per core value (typically 3-7 columns). Each cell holds one of three marks:

- `+` — the person lives this value most of the time
- `+/-` — the person lives this value some of the time and not other times
- `-` — the person rarely lives this value

The three-symbol system is intentionally fuzzy. Core Values are cultural behaviors observed over months, not binary attributes. The `+/-` mark exists specifically to capture "depends on the day" reality without forcing a false yes or no.

#### Right side — GWC columns (two-symbol scoring)

Three columns: G, W, C. Each cell holds one of two marks:

- `Y` — yes, this person does Get it / Want it / has Capacity
- `N` — no

**There is no `+/-` allowed in GWC columns.** GWC is deliberately binary: "mostly" is treated as a no. Ambiguity on seat fit is exactly the disease the tool exists to cure. Two-out-of-three is not a partial pass — it is a fail.

### The bar — minimum acceptable score

Every leadership team using the People Analyzer sets a **bar** — the minimum score for anyone in the company. Wickman's recommended bar for a company with 5 core values:

- 3 `+` marks
- 2 `+/-` marks
- 0 `-` marks
- `Y` on all three of G, W, C

Not a universal law — a starting point. Some teams set higher (4 `+`, 1 `+/-`); a few set slightly lower while calibrating. Once set, every manager knows what acceptable looks like. "Below the bar" becomes a factual claim, not an opinion. Set once, written down, applied consistently across the whole company.

### How the grid is used — continuous, not annual

The People Analyzer runs on a continuous cadence:

- **Quarterly reviews** — every direct report of every leader scored on the grid
- **Hiring decisions** — score the candidate before extending the offer; the bar applies
- **Termination decisions** — review the person's current scores against the bar before the conversation
- **Promotion decisions** — score against the new seat using GWC; Core Values scores carry over

The grid lives in the company's people-management system or a simple shared document. Updated as observations accumulate. NOT a once-a-year performance review artifact.

### What follows a below-the-bar score

A below-the-bar score does not trigger immediate termination. It triggers the Three-Strike Rule — a structured three-round conversation over ~90 days that gives the person specific feedback on the gap, clear expectations, and time to change. Most people either rise above the bar within the first or second strike, or self-select out before the third strike because the process makes the mismatch obvious to them too.

## Key Takeaway
Two scoring systems on one grid: Core Values (+/+-/-) on the left, GWC™ (Y/N) on the right. NO +/- in GWC columns. Bar set once, applied consistently. Runs continuously — every hire, every fire, every quarterly review.
