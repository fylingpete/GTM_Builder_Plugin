## Core Values Discovery — EOS™ (Gino Wickman)

Core values are the small set of timeless, non-negotiable principles that describe how a company actually operates. They are the first answer on the V/TO™ and the backbone of the People Analyzer™. The deepest mistake leadership teams make is to *invent* values by sitting in a conference room and brainstorming abstract virtues — producing a list of generic single-word abstractions that nobody can operationalize.

The book's counter-method: *discover* values rather than invent them, by running a four-step exercise anchored in real humans the team already works with.

### What core values are (and aren't)

Core values are descriptive, not aspirational. They name what the culture already is at its best, not what leadership wishes it were. The test is not "does this sound inspiring" but "can two reviewers look at the same employee's behavior and agree on whether this value was lived or violated."

Required properties:
- **Timeless** — still a core value in 5-10 years
- **Non-negotiable** — company refuses to compromise even when profitable to do so
- **Operational** — usable as a filter for hire/fire/review/reward without ambiguity
- **Already true** — current people are living it, not just talking about it

Anything failing one of these tests is marketing copy, not a core value.

### The 3-7 rule

Target range: 3-7 values. Fewer than 3 under-specifies the culture. More than 7 dilutes the list — beyond 7, no employee can remember all of them. Within the range, smaller is preferred.

### The 4-step clone exercise

**Step 1 — Each leader names three people they would clone.** Privately, before group discussion, each leader writes three names. Strong preference for internal people whose behavior they have actually observed. Privacy prevents the first speaker from anchoring everyone else.

**Step 2 — As a group, list traits those people share.** The only step that looks like brainstorming. Goal: get everything on the board, not edit yet. Duplicates fine, near-synonyms fine.

The reason this works: the team is no longer reaching for abstract virtues — they're looking at specific humans they already trust and asking "what makes these people so valuable." Output is grounded in observed behavior.

**Step 3 — Narrow to 5-15 candidate values.** Editing begins. Combine duplicates, merge synonyms, cross out the non-essential, elevate items that keep coming up.

**Step 4 — Debate down to 3-7 truly core values.** The hardest step. Argue each candidate against the test criteria (timeless / non-negotiable / operational / already true) and keep, drop, or merge. A value no leader is willing to defend out loud is not core. If the team can't agree on a value, the right answer is usually to drop it, not add it.

### Why the clone exercise works

It inverts the usual brainstorm pathology. A normal values brainstorm asks "what SHOULD our values be" — producing abstract virtues every company could claim. The clone exercise asks "who are the people we already trust, and what makes us trust them." Answer is necessarily concrete.

Second benefit: it surfaces values the company *actually has* rather than the values leadership *wishes* it had. If nobody can think of an internal person who embodies "innovation," innovation is not a current core value.

### The operationalization test

Before any candidate gets onto the final list:

> Could two reasonable leaders look at the same employee's behavior over the last quarter and reach the same conclusion about whether this value was lived or violated?

If yes → operational. If no (two reviewers draw different conclusions from the same evidence) → too vague to do any work. The most common failure: single-word abstractions ("Integrity," "Excellence") that mean different things to different people. Fix: expand the value into a phrase that names the specific behavior.

### Why values must be operationalizable

Vague values cause cascading damage because they get used inside other tools that depend on being checkable:
- People Analyzer™ grades each person against each value with +, -, +/-. Vague values produce inconsistent grades.
- Hiring/firing decisions citing values become legally and morally fragile if values can't be applied consistently.
- Review conversations get dragged into definitional disputes instead of focusing on behavior.

Vague values are worse than no values — they create the illusion of a filter without actually filtering anything.

## Key Takeaway
3-7 values. Discovered via the 4-step clone exercise, not brainstormed. Each must pass the two-reviewers-agree operationalization test. Single-word abstractions need expansion into behavioral phrases. Used in real hire/fire/review decisions or the values aren't real.

## When to Apply
- Step 3.2 People Component work, as a first draft to feed the People Analyzer™
- Step 3.2 V/TO™ session, as formal answer to question 1
- Whenever the leadership team's hiring/firing decisions feel arbitrary
- Annual Planning Session, to confirm the existing values still describe the culture (not to rewrite)
